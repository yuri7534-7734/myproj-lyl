CSRF(Cross-Site Request Forgery)
사이트간 요청 위조 - 해커가 사용자 몰래 악의적인 요청을 보내는 공격이다.
1. 정상적인 요청을 해서 인증을 획득하고 웹브라우저는 인증을 간직한다.
2. 해커가 만든 악의적인 사이트 접속.
3. 악성 사이트의 숨겨진 폼으로 접속해서 비정상적인 권한 획득한다.

보안 방법
1. 서버가 로그인 페이지 응답시 -> 고유한 CSRF 토큰(문자열) 발급
2. 인증토큰이 발급된 로그인 폼에서 유저가 요청하게 한다.
3. 서버가 토큰을 검증한다, 발급한 토큰과 일치하면 정상처리
